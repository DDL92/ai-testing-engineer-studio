# Pricing Guidance

## When To Charge $300

- Small app or public flow review.
- Limited access.
- One to two critical flows.
- Audit report only or very light starter test coverage.

## When To Charge $500

- Test credentials are available.
- Three to five critical flows.
- Coverage matrix and starter tests included.
- Client expects a clear roadmap and recommendations.

## When To Charge $1,000+

- Multiple workflows.
- Starter framework setup.
- API checks included.
- AI testing audit with several prompt categories.
- Higher-risk SaaS or AI product.

## When To Offer Retainer

- Client has ongoing releases.
- Tests need monthly expansion.
- Bugs can be converted into regression tests.
- Client needs CI failure triage.
- First audit or framework delivery went well.

## When Not To Discount

- Client asks for faster delivery.
- Scope is unclear.
- Access is complex.
- Work includes AI safety or RAG testing.
- Client wants the same deliverables for less.

## Handling Budget Objections

Best response:

"We can reduce the first scope instead of discounting the full deliverable. The smallest useful starting point is a focused audit around your top 2-3 flows."

## Positioning Audit As Low-Risk Entry Offer

The audit helps the client avoid overinvesting in the wrong tests. It produces a roadmap, priorities, and starter coverage before committing to a full framework.
