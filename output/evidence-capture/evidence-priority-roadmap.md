# Evidence Priority Roadmap

## Priority 1

- Playwright
- Evidence Type: playwright-evidence
- Status: Not Yet Implemented
- Why: Functional flow evidence is the closest match to the Studio's Playwright-first QA service model.

## Priority 2

- Screenshots
- Evidence Type: screenshot-evidence
- Status: Not Yet Implemented
- Why: Reviewed screenshots make audit observations easier to verify and client-ready.

## Priority 3

- Lighthouse
- Evidence Type: lighthouse-evidence
- Status: Not Yet Implemented
- Why: Lighthouse can add low-cost performance and accessibility snapshots once capture rules are controlled.

## Priority 4

- Accessibility
- Evidence Type: accessibility-evidence
- Status: Not Yet Implemented
- Why: Accessibility observations can improve audit value when carefully framed as opportunities, not compliance claims.

## Priority 5

- Performance
- Evidence Type: performance-evidence
- Status: Not Yet Implemented
- Why: Performance evidence supports recurring monitoring and retainer conversations after functional evidence is stable.

## Safety Notes

- Do not run Playwright.
- Do not run Lighthouse.
- Do not capture screenshots.
- Do not perform accessibility scans.
- Do not perform performance scans.
- Do not use browser automation.
- Do not use APIs.
- Do not use credentials.
- Do not use external databases.
- Do not invent evidence, screenshots, metrics, results, findings, bugs, vulnerabilities, or incidents.
