# Real Lead Readiness Checklist

Use this checklist before treating a lead as ready for outreach, audit, or proposal work.

## Lead Data

- [ ] Lead added to `data/leads.json`.
- [ ] Company name is correct.
- [ ] Website is a valid public URL.
- [ ] Industry is recorded.
- [ ] Source is recorded.
- [ ] Notes explain why the lead was selected.
- [ ] Pain points are based only on manual research or local notes.
- [ ] Score was generated.
- [ ] Recommended offer was generated.

## Research Outputs

- [ ] Research pack generated with `npm run lead:research -- --id lead_id`.
- [ ] Lead pack generated with `npm run lead:pack -- --id lead_id`.
- [ ] Outbound plan generated and reviewed.
- [ ] Suggested outreach channel reviewed.
- [ ] Manual next action is clear.

## Audit And Proposal Readiness

- [ ] Audit generated if appropriate.
- [ ] Audit screenshots and findings reviewed.
- [ ] SOW generated if appropriate.
- [ ] Pricing reviewed manually.
- [ ] No unsupported audit findings were added.
- [ ] No fake metrics or invented claims were added.

## Outreach Safety

- [ ] No auto outreach was used.
- [ ] No API, scraping, CRM, or email automation was used.
- [ ] Contact information was manually researched.
- [ ] Message draft was reviewed.
- [ ] Daniel approved the message before sending manually.
- [ ] Follow-up date is tracked manually after a message is sent.

## Final Human Review

- [ ] Lead is still a fit.
- [ ] Offer is appropriate.
- [ ] Next step is bounded and realistic.
- [ ] Client-facing language is professional and cautious.
- [ ] No credentials, secrets, or private client data are included.
