# Follow-Up Library

Follow-ups should be professional, low-pressure, and manual. Do not automate follow-ups or send messages without approval.

## Follow-Up #1

Use 3-5 business days after the first message if there is no response.

```text
Hi {{firstName}}, quick follow-up on the QA audit idea.

The simplest version is a focused review of one public workflow with screenshots, cautious findings, and a few Playwright smoke test recommendations.

Would that be useful to see?
```

## Follow-Up #2

Use 5-7 business days after Follow-Up #1 if the lead still looks qualified.

```text
Hi {{firstName}}, one more follow-up.

If {{company}} is shipping changes regularly, a small QA audit can help identify the first workflow worth protecting with Playwright tests.

If now is not the right time, no problem.
```

## Breakup Message

Use when there is still no response.

```text
Hi {{firstName}}, I will close the loop here.

If QA automation or release confidence becomes a priority later, I am happy to help with a focused audit or a small Playwright Starter Pack.

Wishing you a smooth release cycle.
```

## Follow-Up Rules

- Do not guilt or pressure the lead.
- Do not imply urgency that does not exist.
- Do not invent audit findings.
- Do not mention private data.
- Stop after the breakup message unless the lead replies.
