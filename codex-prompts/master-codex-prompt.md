# Master Codex Prompt

## Audit Web App

Uso: Analyze a web app before automation.  
Prompt: Act as a Senior QA Automation Engineer. Analyze this app flow, identify critical risks, propose 5-10 Playwright smoke scenarios, and recommend stable locators using getByRole/getByLabel first.  
Resultado esperado: Risk-based test plan and automation priorities.  
Notas: Keep scope small and client-deliverable.

## Create QA Strategy

Uso: Build a concise QA strategy.  
Prompt: Create a practical QA automation strategy for this SaaS app using Playwright, TypeScript, POM, fixtures, API health tests, HTML report, and GitHub Actions. Include what not to automate yet.  
Resultado esperado: Delivery-ready strategy.  
Notas: Prioritize first paid delivery.

## Prepare Handoff

Uso: Final client handoff.  
Prompt: Create a client handoff document explaining what was delivered, how to run tests, how to adapt locators, current limitations, and recommended next steps.  
Resultado esperado: Client-ready handoff.
